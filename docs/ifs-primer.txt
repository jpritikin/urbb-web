IFS Simulator Architecture Primer
==================================

Strictly enforce model-view separation — playback failures are usually violations of
this. Model state that depends on timers must use simulation time, not wallclock time.

Core pattern: MVC with act/sync flow.
All state mutations go through cloudManager.act(action, fn):
  1. Clones model (for diff)
  2. Runs fn() which mutates model
  3. Calls syncViewWithModel(oldModel)
  4. Records action for playback

Dialogue Writing Guidelines
----------------------------
Dialogues flash by fast with minimal context. Write them to make sense standalone:
- Spell out referents; avoid "it", "this", "that" without clear antecedent.
- Prefer specific language ("Self is asking about my job") over vague ("Self wants to know").
- Keep lines short enough to read at a glance.

Modes
-----
SimulatorMode: 'panorama' | 'foreground'
- Panorama: clouds on torus, click selects target (enters foreground).
- Foreground: conference table. Targets at seats, blended at star, supporting parts orbit target.
- Transitions via TransitionAnimator. Clouds move between zoomGroup and uiGroup.

Action Flow
-----------
1. User clicks cloud/star → inputHandler routes based on mode
2. Pie menu shows valid actions (simulatorController.getValidActions())
3. User selects → pieMenuController dispatches to cloudManager handler
4. cloudManager.act(label, () => {
     result = controller.executeAction(actionId, cloudId, options);
     effectApplicator.apply(result, cloudId);
   })
5. act() calls syncViewWithModel()

Internal actions: ray_field_select, spontaneous_blend, backlash, mode_change,
process_intervals. All in actionFormatter's KNOWN_ACTIONS.

Pending Actions
---------------
feel_toward, notice_part, add_target need a second click. Controller sets
model.pendingAction = { actionId, sourceCloudId }. View shows banner with dismiss.
Next cloud click → cloudManager.completePendingAction(). Cleared on dismiss,
mode change to foreground, or completion.

Blending Lifecycle
------------------
spontaneous: needAttention threshold → partDemandsAttention() → addBlendedPart
  - Already a target: removes from targets first.
  - Not in conference: displaces existing targets (displacedParts, spiral exits).
  - Blended needAttention decays (0.98^dt).
therapist: blend action → removeTargetCloud + addBlendedPart
Separation: separate → reduceBlending → degree→0 → promoteBlendedToTarget →
  syncViewWithModel() → completedUnblendings → stretch-resolution animation (view only)
Pending blends: enqueuePendingBlend → view animates arrival → promotePendingBlend

Conference Participants
-----------------------
model.getConferenceCloudIds() = union of targets + blended + pending blends + supporting parts.

Conversation System
-------------------
Authoritative spec for game mechanics: ~/coach/src/ifsConversationSim.ts
Dialogue writing guide: ~/coach/docs/conversation.txt
Analysis tool: npx tsx scripts/extractConversationData.ts <recording.json>

model.isConversationPossible(): requires foreground, no blended parts, exactly 2 targets
with an inter-part relation.
model.initConversation(): applies stance flips, picks initial speaker (highest stance),
sets phases (speaker→speak, listener→listen).

Effective stance = relation stance + therapistDelta + shockDelta (clamped to [-1,1]).
Therapist and shock deltas decay exponentially.

IFIO phase cycle:
- 4-step: speak/listen → listen/mirror → validate/listen → listen/empathize
- 6-step: adds clarify/listen → listen/mirror_again between mirror and validate
Cycle length from tuple length, picked once at cycle start (pickTupleIndex()).
All lines — speaker and listener — come from the speaker's tuple.
Phases advance when regulationScore > 0.5 and respondTimer elapses.
On empathize: trust bonus to both, speaker's stance softened. Both enter listen/listen;
after NEW_CYCLE_DELAY the highest-stance part becomes new speaker.

Dysregulated speaker: probabilistic utterance that doesn't advance phase; sets
dysregulatedSpokePending until regulation recovers, then phase advances.
SpeakRole in listen phase: fires dysregulated utterance after LISTENER_VIOLATION_GRACE.
ListenRole violation: passive listener exceeds REGULATION_STANCE_LIMIT for >1s →
resets conversation with that part as new speaker.
Polarity flip: receiver shocked below -REGULATION_STANCE_LIMIT with p=stanceFlipOdds →
erupts, becomes new speaker.

Stance shock: streakMult × 0.3 × |speakerStance| × 2 / ((1+selfTrust)(1+interTrust)).
Default polarity: opposite to speaker. With p=stanceFlipOdds: same as speaker.
streakMult = e^(k×streak), k=log(1.2)/5.

Backlash
--------
checkBacklash() iterates ALL non-consenting protectors, reduces each trust by trustGain/2,
probabilistically triggers backlash (newTrust < rng). Multiple triggers → one picked randomly.

Notice_part on a protector: if noticer's trust > protector's trust, sets inter-part
trustFloor to 0.25 (impactRecognition dialogues); otherwise impactRejection dialogues.

Validate requires an unvalidated thought bubble (extends expiry on validation).
Gratitude on active protectors requires jobAppraisal revealed first.

View: star border morphs for active IFIO conversation. Carpets drag-rotatable to nudge
therapist delta. Phase labels and stance shown as carpet badges.
